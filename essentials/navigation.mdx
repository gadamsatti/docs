## Documentation Structure

Our documentation is organized into the following sections:

### Getting Started

- Introduction to EpicMarket
- Quickstart guide
- Platform overview

### Business Guide

- Setting up your business
- Managing your storefront
- Handling bookings
- Analytics and reporting

### Customer Guide

- Using the mobile app
- Finding businesses
- Making bookings
- Payment methods

### API Reference

- Authentication
- Endpoints
- Webhooks
- SDKs

## Navigation Example

Here's how to structure your navigation in `mint.json`:

```json
{
  "navigation": [
    {
      "group": "Getting Started",
      "pages": ["introduction", "quickstart"]
    },
    {
      "group": "Business Guide",
      "pages": [
        "business/setup",
        "business/storefront",
        "business/bookings",
        "business/analytics"
      ]
    },
    {
      "group": "Customer Guide",
      "pages": [
        "customer/mobile-app",
        "customer/discovery",
        "customer/bookings",
        "customer/payments"
      ]
    },
    {
      "group": "API Reference",
      "pages": [
        "api-reference/introduction",
        {
          "group": "Endpoints",
          "pages": [
            "api-reference/endpoint/create",
            "api-reference/endpoint/get",
            "api-reference/endpoint/delete"
          ]
        }
      ]
    }
  ]
}
```

<Note>
  Make sure all referenced pages exist in your documentation folder structure.
</Note>
