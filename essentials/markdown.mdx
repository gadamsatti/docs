## Writing Style

When documenting EpicMarket features, follow these guidelines:

### Headers

Use headers to organize content hierarchically:

```md
## Major Sections

### Subsections

#### Detailed Points
```

### Formatting

Use markdown formatting to enhance readability:

| Style  | Syntax        | Example                                       |
| ------ | ------------- | --------------------------------------------- |
| Bold   | `**text**`    | **Important note**                            |
| Italic | `*text*`      | _Additional info_                             |
| Code   | `` `code` ``  | `apiKey`                                      |
| Links  | `[text](url)` | [Dashboard](https://dashboard.epicmarket.com) |

### Code Examples

Include clear code examples with language specification:

```javascript
// Initialize EpicMarket SDK
const epicMarket = new EpicMarket({
  apiKey: "YOUR_API_KEY",
});
```

### Callouts

Use callouts to highlight important information:

<Note>Important information about using EpicMarket features.</Note>

<Warning>Critical warnings about potential issues or requirements.</Warning>

<Tip>Helpful tips for getting the most out of EpicMarket.</Tip>

### Lists

Organize information in lists:

- Unordered lists for general points
- Ordered lists for sequential steps
  1. First step
  2. Second step
  3. Third step

### Tables

Use tables to compare features or display structured data:

| Plan       | Price  | Features               |
| ---------- | ------ | ---------------------- |
| Basic      | $10/mo | Core features          |
| Pro        | $30/mo | Advanced features      |
| Enterprise | Custom | All features + Support |

### Images

Include screenshots or diagrams when helpful:

```md
![Feature Screenshot](/images/feature.png)
```

<Note>Always compress images and provide alt text for accessibility.</Note>
